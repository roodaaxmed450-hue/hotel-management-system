KHEYRE HOTEL MANAGEMENT SYSTEM (KHMS)

Single Django App – Full Requirements

1. System Overview

Kheyre Hotel Management System (KHMS) is a web-based internal hotel management system built using one Django app only.
The system manages all daily hotel operations from a single, unified application.

Technology stack:

Backend: Django

Frontend: Django Templates + Tailwind CSS + JavaScript

Database: Django default SQLite (initial)

2. Core Design Rule (VERY IMPORTANT)

The entire system must be implemented inside ONE Django app

No multiple apps (no accounts/, rooms/, bookings/, etc.)

All models, views, URLs, and templates belong to one app

Django’s built-in authentication system is used

Permissions are handled using roles inside the same app

3. User Roles & Access Control
3.1 System Roles

Only two roles exist:

Admin / Hotel Owner

Receptionist

No cleaners, no accountants, no managers.

3.2 Role Permissions
Admin / Owner

Full system access

Create, update, activate, deactivate receptionist users

Manage rooms and room pricing

View all bookings, guests, payments, and expenses

Approve or reject expenses

View all reports

Access full dashboard

Receptionist

Register and manage guests

Create and manage bookings

Perform check-in and check-out

Generate invoices

Record payments (cash / mobile money)

Record expenses (requires admin approval)

View limited daily reports

4. Authentication & Login Flow

Django built-in authentication (django.contrib.auth)

All users log in using username and password

After login:

Admin → Admin Dashboard

Receptionist → Receptionist Dashboard

Views and templates must restrict access by role

5. User Management (Inside Same App)

Admin creates receptionist users from the system

Each user has:

Username

Password

Role (Admin or Receptionist)

Status (Active / Inactive)

Inactive users cannot log in
